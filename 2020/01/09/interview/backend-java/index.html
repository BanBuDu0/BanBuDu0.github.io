Java HashMap

HashMap初始化长度为16

HashMap初始化的长度是使用位运算计算出来的

如果HashMap的某一index的链表长度超过8，static final int TREEIFY_THRESHOLD = 8;则会自动把链表转换为红黑树加速插入和查找

HashMap 扩容默认负载因子 = 0.75；redis没有BGSAVE和BGREWRITEAOF的时候默认负载因子=1的时候扩容，有的时候负载因子=5的时候扩容

为什么重写equals()方法后必须对hashCode()方法进行重写？

因为equals方法返回相等的两个对象他们的hashCode方法返回的值也必须相等，反之则不一定。
在上述约定下，如果重写了equals而没重新hashCode，就会导致equals相等的两个对象的hashCode不相等。这会造成的问题是，比如在HashMap中，首先是根据Key的hashCode来判断其在数组中的index，再对该index上的链表/红黑树上的元素判断equal，所以可能会造成两个equals相等的key，在HashMap里面存了两份不一样的value（但其实应该要后Put的覆盖前面的value）。

Java8 运行时常量池在哪里

字符串常量池在Java7之前保存在方法区，Java8之后保存在堆上。其他运行时常量池Java7之前保存在方法区，Java8之后保存在元空间（Java8方法区的实现变成了元空间）。
