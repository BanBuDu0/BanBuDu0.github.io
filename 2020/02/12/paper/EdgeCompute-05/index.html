边缘智能 EI

文章将EI划分为6个等级，如下所示

  1. Cloud–Edge Coinference and Cloud Training: DNN模型在云端训练，但以云边协作方式推理DNN模型
  2. In-Edge Coinference and Cloud Training: DNN模型在云端训练，但在边缘推理DNN模型
  3. On-Device Inference and Cloud Training: DNN模型在云端训练，但在设备推理DNN模型
  4. Cloud–Edge Cotraining and Inference: 云边协同训练和推理DNN模型
  5. All In-Edge: DNN模型训练和推理都在边缘端
  6. All On-Device: DNN模型训练和推理都在设备端

随着EI level 的提高，数据卸载的数量和路径长度减少。从而降低了数据转移的传输延迟，增加了数据的保密性，降低了广域网带宽的开销。然而，这是以增加计算延迟和能源消耗为代价实现的。

边缘智能模型推理

从架构、关键性能指标、支持技术以及现有的系统和框架四个方面来阐述

架构

架构方面主要可以分为基于边缘、基于设备、边缘-设备、边缘-云四种DNN推理模式

1. 基于边缘

设备获取数据发送到边缘，在边缘服务器上进行DNN模型推理，再将预测结果发回设备

主要的缺点是推断性能依赖于设备和边缘服务器之间的网络带宽

2. 基于设备

在设备本地进行模型推理

但是需要大量的本地资源，性能取决于本地设备本身

3. 边缘-设备

将DNN模型划分为多个部分，分别在边缘和设备上执行特定的层

与基于边缘和基于设备的模式相比，边缘设备模式更加可靠和灵活。它也可能需要巨大的资源

4. 边缘-云

类似于边缘-设备模式，适用于设备资源高度受限的情况云边

模型的性能在很大程度上取决于网络连接质量

关键性能指标

有如下五个性能指标

  1. latency
    延迟受许多因素的影响，包括边缘设备上的资源、数据传输方式和DNN模型的执行方式
  2. accuracy
    除了DNN模型自身的推理能力外，推理精度还取决于将输入数据输入DNN模型的速度
  3. energy
    能量消耗取决于DNN模型的大小和边缘设备上的资源的影响
  4. privacy
    隐私保护取决于原始数据的处理方式
  5. communication overhead
    通信开销主要取决于DNN推理方式和可用带宽
  6. memory footprint
    内存占用主要受原始DNN模型的大小和加载大量DNN参数的方式的影响

支持技术

介绍八种用于改进上述一个或多个关键性能指标的支持技术

1. Model Compression 模型压缩

模型压缩优化了延迟，能源，隐私和内存占用四个指标。模型压缩技术包括weight pruning权重修剪, data quantization数据量化, compact architecture design紧凑的体系结构设计。

权重修剪从训练的DNN中去除多余的权重

数据量化不采用32位浮点格式，而是使用更紧凑的格式来表示层输入、权重或两者

AdaDeep系统地将精度、延迟、存储和能量等方面的目标和约束规定为一个统一的优化问题，并利用DRL有效地找到压缩技术的良好组合。《On-demand deep model compression for mobile devices: A usage-driven model selection framework》

2. Model Partition 模型划分

模型划分如下图所示，是将DNN模型的计算密集部分卸载到边缘服务器上，主要关注延迟、能量和隐私等问题。

模型划分可以分为两类：服务器-设备划分、设备-设备划分

3. Model Early Exit 模型提前退出

延迟是模型提前退出的优化目标。模型提前退出如下图所示，特定的层位置添加出口分支来修改标准的DNN模型结构

4. Edge Caching 边缘缓存

通过缓存DNN推理结果来优化延迟问题，边缘缓存的核心思想是在网络边缘缓存和重用任务结果

5. Input Filtering 输入过滤

输入过滤的关键思想是去除输入数据的非目标对象帧，避免DNN模型推理的冗余计算。提高推理精度，缩短推理延迟，降低能量消耗

6. Model Selection 模型选择

优化DNN推理的时延、精度和能量问题。模型选择的主要思想是先离线训练一组DNN模型，然后自适应地在线选择模型进行推理

7. Support for Multitenancy 支持多组织

端部或边缘设备通常同时运行多个DNN应用程序，多个DNN应用程序将争夺有限的资源。支持多组织侧重于优化全局能源和内存占用

8. Application-Specific Optimization 特定于应用程序的优化

优化EI应用程序的准确性，延迟，能耗和内存占用，对于特定的应用程序特定优化，如对于基于视频的应用，可以灵活地调节帧速率和分辨率，以减少资源需求。
