软件超密集网络中移动边缘计算的任务卸载

  • Hessian matrix 黑塞矩阵
    黑塞矩阵是由一个多元函数的二阶偏导数构成的方阵，描述了函数的局部曲率，常用于牛顿法解决优化问题，利用黑塞矩阵可判定多元函数的极值问题。
  • 线性与非线性
    线性表示两个变量之间是一次函数的关系，线性关系中的量是成比例的
    非线性即除了两个变量是一次函数关系之外的关系
  • Karush-Kuhn-Tucker conditions
    KKT 条件是在满足一些有规则的条件下，一个非线性规划问题能有最优解的充要条件

移动边缘计算主要展现了两个优点

  1. 减少了任务持续时间

    • 任务上传到云端的时间
    • 在云端执行的时间
    • 云端返回结果的时间

  2. 减少了移动设备的能量消耗

    • 延迟敏感性计算应该放在边缘端执行，没有延迟要求的计算可以放在云计算中心执行
    • 大体量但是仅需少量的计算资源的任务应放在用户设备执行

software defined ultra dense network(SD-UDN)

软件定义的超密集网络
核心思想是分离控制层和数据层（通过虚拟化技术）
在未来的通信网络中，边缘节点将被部署在超密集网络中为用户提供计算卸载服务

software defined network(SDN) 软件定义网络

利用OpenFlow协议将路由器的控制平面从数据平面中分离出来，改用软件的方式实现。该架构可使网络管理员在不变动硬件设备的前提下，以中央控制方式用程序重新规划网络。

移动边缘计算在软件超密集网络中的有效应用将会面临的挑战

1. 如何控制和分配少量的分布式资源

  • 小蜂窝基站的密集部署增加了网络环境的复杂性
  • 边缘端的计算资源是有限的

2. 如何根据手机剩余电量和网络状态来实施计算任务卸载到边缘端

  • 用户对他们加入的无线网络只有很少的信息，不知道一些关键信息例如——网络的流量负荷和边缘端的计算负荷等信息

移动边缘计算主要有两个研究点

  1. 任务卸载策略
  2. 资源分配策略

创新点

1. 提出了软件定义的超密集网络架构(SD-UDN)

当前一些移动边缘计算的研究一般只考虑单一的移动边缘计算服务

考虑到了多服务器的又没有考虑到超密集网络

基于控制域和数据域的分离。将controller部署在宏蜂窝基站，根据controller的决策，移动设备可以决定是将计算任务卸载到边缘端执行还是在本地执行。

A. SD-UDN 包含3层结构

  1. 用户层

哪些用户需要卸载计算任务
用户通过无线网络和小蜂窝基站或者宏蜂窝基站连接
本文忽略了移动设备向控制者发送数据时的时延和能耗
2. 数据层

小蜂窝基站和部署在小蜂窝基站附近的边缘节点
小蜂窝基站通过高速前沿运输网络和宏蜂窝基站连接
3. 控制层

部署在宏蜂窝基站的controller
完成主要的控制功能，如资源分配调度，信息汇总等

D. 任务卸载模型

给出一些公式
本地计算
本地执行时间 = 任务计算量 / CPU能力
本地执行耗能 = 发射功率 x 任务计算量
移动边缘计算
任务持续时间 = (任务计算量 / (计算资源 x 分配到的计算资源的比例)) + (计算任务的尺寸大小 / 数据上传到边缘端的传输速率)

忽略了边缘端返回计算结果的时延

其中 Xi 是决策0-1变量，决定任务是在本地执行还是边缘端执行，最优化问题的解就是本地执行和边缘执行的所有任务持续时间在约束条件下的最小值，约束条件如下：

  1. 本地耗能比电池余量要小

  2. 任务传输到边缘端的耗能比电池余量要小

  3. 执行任务的边缘节点是提供服务的节点
    Yi是分配给任务Qi的边缘节点，A(ui)是提供服务的基站

  4. 所有计算资源的分配不能超过该节点的计算资源总和

2. 软件定义的任务卸载模型(software defined task offloading-SDTO)，不仅考虑到了如何减少任务持续时间而且考虑到了电池能力

该模型包含以下两个方面

  • 计算任务在哪里执行——本地或是边缘端，哪个边缘（任务分配问题）
    将该问题转化为0-1优化问题
    但是这里没有考虑云计算中心的任务卸载私认为可以将一些计算密集型且非延迟敏感性任务卸载到云计算中心

  • 边缘应该给每个任务分配多少资源（资源分配问题）
    将该模型转化为凸优化问题

  • 证明了计算资源分配是是凸优化问题

  • 给出了任务分配算法

  • 进行了大量的实验证明SDTO模型比随机模型和统一任务计算卸载模型更好

定义该模型为混合整数非线性规划模型且证明是NP难问题

因为任务卸载是整体的任务卸载，所以是整数规划问题

该模型的目标是在电量消耗这个条件下最小化任务持续时间
