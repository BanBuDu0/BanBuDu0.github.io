增强车辆边缘计算网络中的用户体验：一种自适应资源分配方法

现状

  • service reliability (i.e., losing less data)
  • freshness (i.e.,as quickly as possible)

以上两点不能共存

可拓展的计算任务结构 - 采用了两种计算质量转换

  1. configurations substitute
    使用简单的计算任务代替复杂的计算任务
  2. data drop allocation
    丢弃某些冗余任务. 如果通过任务的子集实现了用户指定的准确性, 则应用程序可以选择跳过剩余的任务并尽早终止.

这样做好像有更多的网络资源去执行任务, 但是这是以计算质量为代价的.
加入无线卸载策略(无线资源分配)后这种情况更加复杂.
这种情况下应该坚持如下三点:

  1. 计算质量自适应确定每个计算任务的计算配置选择和数据丢失
  2. 无线电资源分配控制无线传输速率，以将数据从车辆卸载到移动边缘计算服务
  3. 任务放置问题根据各种移动边缘计算服务的可用性和负载来决定要放置任务的边缘节点。

这三点是高度耦合的，应该同时考虑。又因为车辆的动态性，保证长期计算质量的资源分配需要动态适应性。

相关工作

  • 设计了一种自适应资源分配方法
  • 制定了一种在线算法（Lyapunov优化），将问题分解为一系列的on-slot优化问题；提出一种基于迭代的方案（DC规划）解时隙问题（混合离散优化问题）
  • 自适应性实现不需要任何关于未来网络状态信息的先验知识，并且所提出的算法计算复杂度低

提出了ARAEUE(Adaptive Resource Allocation for Enhancing User Experience)在线算法

adaptively adjust the follow allocation problems to minimize the long-term computing quality loss while balancing the queuing delay

延迟敏感型任务在有严格deadline的等待队列等待数据

  1. TP - task placement 任务分配
  2. RR - radio resource 无线资源
  3. CC - computation configuration 计算配置
  4. DD - data drop 数据丢弃

Network Model - 网络模型

其中 $l_k \in \mathcal L_k$ 量化任务$K$的精确计算结果的近似最优，以处理运行时产生的不确定性，表示对计算精度损失的最大容忍度。$l_k$ 越低，用户体验越好。
模型采用了SDN技术，在优化系统中引入了全局网络状态的中心控制器，实现了对分布式边缘节点和移动车辆的逻辑集中控制

边缘计算模型

每个车辆的每个任务都有不同的队列缓冲区，队列采用先进先出。并为每个任务定义了QoS（服务质量）对延迟进行约束，如果到达数据超过了延迟边界就将其删除。

所有队列缓冲区都是无限的，并且所有队列在最初时间 $t = 0$ 时都是空的，队列采用FIFO。每种类型的计算任务定义了服务质量（QoS）来约束延迟要求，超过延迟的直接丢弃。
