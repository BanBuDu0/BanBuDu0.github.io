软件超密集网络中移动边缘计算的任务卸载

  • Hessian matrix 黑塞矩阵
    黑塞矩阵是由一个多元函数的二阶偏导数构成的方阵，描述了函数的局部曲率，常用于牛顿法解决优化问题，利用黑塞矩阵可判定多元函数的极值问题。
  • 线性与非线性
    线性表示两个变量之间是一次函数的关系，线性关系中的量是成比例的
    非线性即除了两个变量是一次函数关系之外的关系
  • Karush-Kuhn-Tucker conditions
    KKT 条件是在满足一些有规则的条件下，一个非线性规划问题能有最优解的充要条件

移动边缘计算主要展现了两个优点

  1. 减少了任务持续时间

    • 任务上传到云端的时间
    • 在云端执行的时间
    • 云端返回结果的时间

  2. 减少了移动设备的能量消耗

    • 延迟敏感性计算应该放在边缘端执行，没有延迟要求的计算可以放在云计算中心执行
    • 大体量但是仅需少量的计算资源的任务应放在用户设备执行

software defined ultra dense network(SD-UDN)

软件定义的超密集网络
核心思想是分离控制层和数据层（通过虚拟化技术）
在未来的通信网络中，边缘节点将被部署在超密集网络中为用户提供计算卸载服务

software defined network(SDN) 软件定义网络

利用OpenFlow协议将路由器的控制平面从数据平面中分离出来，改用软件的方式实现。该架构可使网络管理员在不变动硬件设备的前提下，以中央控制方式用程序重新规划网络。

  • 用户对他们加入的无线网络只有很少的信息，不知道一些关键信息例如——网络的流量负荷和边缘端的计算负荷等信息

方法

交通预测问题

  1. 将传感器网络表示为带权有向图 G = (V, E,W)，其中 V 是点集， E 是边集， W 是 N * N 的带权邻接矩阵表示节点的邻近度

  2. 将在图 G 上观察到的交通流量视为一个 N * P 的图信号矩阵 X ，其中 P 是每个节点上的特征数，特征例如速度、体积等。 X 就是整个路网在某一时刻的状况

  3. 用 X(t) 表示 t 时刻观测到的路网状况

  4. 交通预测的问题就在于学习一个函数 h(.)，通过过去的路网状况矩阵 T’，预测未来 T 时刻的路网状况矩阵。

将交通流建模为扩散的过程

  • random walk 随机游走
    随机游走是指基于过去的表现，无法预测将来的发展步骤和方向
    一般的随机游走是以马尔可夫链或者马尔可夫过程的形式出现的

  • 马尔可夫性质如下

这个扩散过程的特征是在图 G 上重启概率 $\alpha$ 在[0, 1]之间和包含转移矩阵$D_O^{-1}W$的随机游走。其中$D_O$是表示图中顶点出度的对角矩阵

  • sequence to sequence architecture 序列到序列模型
    使用两个循环神经网络，将一个序列直接转换到另一个序列。一般通过Encoder-Decoder（编码-解码）框架实现

  • Autoregressive 自回归
    从线性回归发展而来，不是用来预测其他变量，而是用来预测自己，所以叫自回归
